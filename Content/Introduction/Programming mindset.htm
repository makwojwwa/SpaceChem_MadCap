Programming mindset

SpaceChem applies a visual programming language, which means that you don't have to write code to create algorithms. All commands (called instructions A building block for algorithms in reactors. Must be placed inside the reactor along the path of a waldo so that it can be executed. in-game) are in the form of symbols, which are placed on the grid in a desired sequence. Thanks to that, the game is relatively easy to grasp for a person without any programming knowledge and experience.

Bear in mind that you will not learn how to program by playing SpaceChem. However, by solving successive levels you have an opportunity to acquire a programming mindset, which might help you become a programmer in the future, if you wish so.

What you can learn

  • Every solution is based on inputs and outputs.
    The input is what you receive from outside (e.g. data entered by a website user while logging in) and the output is what your program produces based on the input and the instructions A building block for algorithms in reactors. Must be placed inside the reactor along the path of a waldo so that it can be executed. you add.
  • You can get different inputs and have to create different solutions.
    You have to process the input such that the required output is obtained. That is why if you get two different atoms in the same input area, you have to recognize them and create separate algorithms so that they can be processed in the required manner. When using the real life example mentioned above, your program used by the website has to recognize the first name entered by the user and know what to do with this particular piece of information so that the user can log in.
  • You might require more than one input to produce required output.
    Sometimes you need two different atoms coming from two different input areas to produce a required molecule. In other words, you need a username and a password entered by a user so that the user can be logged in by your program. You will not achieve your goal if you don't get either of them or if you accidentally apply an algorithm that matches a different input than actually present.
  • Tasks are complex but can be divided into sections.
    A task of creating a molecule (e.g. logging a user in) is complex: it consists of several sections. Handle one section, test if it does what it is supposed to and then proceed to another section. Try to combine all sections such that the required output is created.
  • Tasks can be similar and can be re-used.
    If you know how to bind an oxygen atom with a carbon atom (e.g. a username with a password), you can use this template to create an algorithm for binding an hydrogen atom with a nitrogen atom (e.g. a different username with a different password).
  • Programs are made of loops.
    You get two atoms and you have to create a molecule. You recognize them, apply the sequence of instructions A building block for algorithms in reactors. Your program is ready to get new inputs and create new outputs.
  • Programs can contain branches.
    Sometimes your inputs contain atoms that you don't need or appear in a wrong sequence - this happens in production assignments This is a specialised term used in the game for a game level. There are research, production, and defense assigments.. In this case you can create branches and direct those wrong atoms straight to the recycler. The same can happen in our real-life example: if your program expects a particular input after the first one (a password after a username), it must be prepared for a situation where e.g. a username is entered again instead of a password. A programmer must expect the unexpected and prepare branches to handle such incorrect inputs.
